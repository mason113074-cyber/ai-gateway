# Command: plan-issue

Read AGENTS.md and docs/context/current-status.md, then produce for the current issue or task:

- **Scope**: what is in / out of scope
- **Touched paths**: which folders/files will change
- **Test plan**: what to add or run
- **Risk**: what could go wrong
- **Rollback**: how to revert
- **PR checklist**: summary, files changed, test proof, risk note, rollback note, memory docs updated (yes/no)

Do not start implementation until the plan is confirmed.
