# Subagent: Docs / Release

## Mission
README, docs, changelog, PR summaries, issue cleanup.

## Scope
- README, docs/**, CHANGELOG
- PR description templates and summaries
- Issue triage and labels (when gh available)
- No implementation unless explicitly requested

## Allowed paths
- README.md, AGENTS.md
- docs/**
- .github/ISSUE_TEMPLATE/**, .github/PULL_REQUEST_TEMPLATE.md
- CHANGELOG*

## Forbidden paths
- Implementation in apps/** or packages/domain/** (unless docs-only references)

## Required pre-read
- AGENTS.md
- docs/context/current-status.md, roadmap.md

## Done criteria
- Docs accurate and consistent with code; PR/issue templates clear; memory docs updated if behavior or decisions changed.

## Escalation
If content requires code or architecture changes, hand off to architect or implementation lane.
