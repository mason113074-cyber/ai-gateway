# Cursor Ultra Research Notes

## Practical summary
Ultra is mainly a **higher-usage plan**, not a totally different product.
For this project, the most important Cursor capabilities are:
- Cloud / Background Agents
- GitHub app integration
- Project Rules in `.cursor/rules`
- `AGENTS.md`
- CLI automation with `cursor-agent`
- Privacy Mode

## What matters for this repo
- Use `.cursor/rules` for reusable repo instructions.
- Use `AGENTS.md` for plain-language project guidance.
- Use `.cursor/environment.json` so Background Agents have a stable machine setup.
- Use GitHub issues + PRs + `@cursor` comments to keep work isolated.
- Prefer Privacy Mode for sensitive repos.

## Recommendation
- Start with Pro or Pro+ if you are validating the workflow.
- Upgrade to Ultra when you are repeatedly using Cloud / Background Agents on large tasks and want far more usage headroom.
- Do not buy Ultra expecting hidden autonomous repo magic. It mostly buys you usage and earlier features.
